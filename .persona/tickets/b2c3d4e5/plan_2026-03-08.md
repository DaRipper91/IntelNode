# Implement PRoot Command Builder and Atomic Extraction Implementation Plan

## Overview
Refactor the PRoot command construction to use a builder pattern for better maintainability and implement atomic rootfs extraction to prevent filesystem corruption.

## Scope Definition (CRITICAL)
### In Scope
- Create `ProotCommandBuilder` class in `lib/models.dart`.
- Refactor `Workflow` to use the builder for command generation.
- Modify `Workflow.initForFirstTime` to perform atomic extraction via a staging directory.
### Out of Scope (DO NOT TOUCH)
- Modifying the underlying `proot` or `tar` binaries.
- Changing the default mount points or environment variables (only refactoring how they are represented).

## Current State Analysis
- `lib/workflow.dart:739`: Static `boot` string requires shell-side expansion.
- `lib/workflow.dart:966-988`: Extraction logic writes directly to `containers/0`.
- `lib/models.dart`: Currently only contains simple data classes.
