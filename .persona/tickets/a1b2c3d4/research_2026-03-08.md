# Research: Bootstrap Scripts and Workflow String Audit

**Date**: 2026-03-08

## 1. Executive Summary
This research documents the current implementation of the bootstrap system and shell command construction in the Arch Linux ARM migration. Key findings include hardcoded shell logic in `lib/workflow.dart` and the delivery of critical bootstrap scripts via a compressed tarball asset.

## 2. Technical Context
- **`lib/workflow.dart:739`**: The `boot` static string contains the full PRoot command line, including environment variables and mount points.
- **`lib/workflow.dart:873-930`**: The `setupBootstrap` method handles the deployment of binaries and the `patch.tar.gz` asset.
- **`lib/workflow.dart:964`**: The `initForFirstTime` method contains a hardcoded shell block for rootfs extraction and user/group initialization.
- **`assets/patch.tar.gz`**: Contains `tiny/extra/start-arch.sh` and `tiny/extra/start-desktop`.

## 3. Findings & Analysis
### Bootstrap Delivery
The `patch.tar.gz` asset is extracted in `setupBootstrap` (line 928) using `tar zxf patch.tar.gz`. This asset currently contains:
- `tiny/extra/start-arch.sh`: Bypasses systemd, initializes DNS, and starts the system D-Bus daemon.
- `tiny/extra/start-desktop`: Synchronizes with Termux:X11 and launches the XFCE session.

### Hardcoded Shell Logic
`lib/workflow.dart` contains several large blocks of shell code wrapped in Dart strings:
- **`Workflow.setupBootstrap` (lines 881-930)**: Handles symlink creation for binaries and extraction of the patch archive.
- **`Workflow.initForFirstTime` (lines 966-988)**: Handles rootfs extraction, user/group file modification (`passwd`, `shadow`, `group`), and cleanup.
- **`Workflow.launchCurrentContainer` (lines 1165-1180)**: Exports environment variables and triggers the `boot` command.

### Technical Debt: Lint Warnings
`flutter analyze` identified 8 `duplicate_ignore` warnings for `use_build_context_synchronously` in `lib/workflow.dart` at lines 924, 925, 948, 985, 997, 998, 1039, and 1074.

### VNC Initialization Logic
VNC resolution and password patching are handled via `sed` commands constructed in `initData` (lines 1030-1033) and stored in `G.postCommand`.

## 4. Technical Constraints
- **PRoot Isolation**: The containerized environment requires specific mount points (`/dev`, `/proc`, `/sys`) and `PROOT_NO_SECCOMP=1` for Android 16 compatibility.
- **Asset Size Limits**: Rootfs is split into 98MB chunks (`xa*`) to fit within APK constraints.

## 5. Architecture Documentation
- **Two-Stage Bootstrap**: The system uses `start-arch.sh` as the entry point, which then executes `start-desktop` under the `tiny` user.
- **Asset-to-Data Copying**: Assets are copied from the APK bundle to `G.dataPath` before extraction or execution.
